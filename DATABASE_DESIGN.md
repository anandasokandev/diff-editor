# MSSQL Database Design for Canvas State

This document outlines the schema for storing canvas designs, including the enriched JSON state generated by the Craftly Editor.

## 1. Table: `CanvasDesigns`

This table stores the high-level metadata of each design and the complete JSON blob of its elements.

| Column Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| **DesignID** | `UNIQUEIDENTIFIER` | `PRIMARY KEY`, `DEFAULT NEWID()` | Unique project identifier. |
| **UserID** | `UNIQUEIDENTIFIER` | `NOT NULL` | The owner of the design. |
| **DesignName** | `NVARCHAR(255)` | `NOT NULL` | Name of the design (e.g., "Instagram Post"). |
| **Width** | `INT` | `NOT NULL` | Canvas width in pixels. |
| **Height** | `INT` | `NOT NULL` | Canvas height in pixels. |
| **Background** | `NVARCHAR(100)` | `DEFAULT '#ffffff'` | Canvas background color or gradient string. |
| **ThumbnailUrl** | `NVARCHAR(2048)` | `NULL` | URL to the exported PNG/JPG preview. |
| **ContentJSON** | `NVARCHAR(MAX)` | `NOT NULL` | The enriched JSON blob of elements. |
| **CreatedAt** | `DATETIME2` | `DEFAULT GETUTCDATE()` | Creation timestamp. |
| **UpdatedAt** | `DATETIME2` | `DEFAULT GETUTCDATE()` | Last modification timestamp. |

## 2. SQL Schema Script

```sql
CREATE TABLE CanvasDesigns (
    DesignID UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
    UserID UNIQUEIDENTIFIER NOT NULL,
    DesignName NVARCHAR(255) NOT NULL,
    Width INT NOT NULL,
    Height INT NOT NULL,
    Background NVARCHAR(100) DEFAULT '#ffffff',
    ThumbnailUrl NVARCHAR(2048) NULL,
    ContentJSON NVARCHAR(MAX) NOT NULL,
    CreatedAt DATETIME2 DEFAULT GETUTCDATE(),
    UpdatedAt DATETIME2 DEFAULT GETUTCDATE(),

    -- Ensure ContentJSON is always valid JSON
    CONSTRAINT [CK_CanvasDesigns_ContentJSON_IsJSON] 
    CHECK (ISJSON(ContentJSON) > 0)
);

-- Index for User lookup
CREATE INDEX IX_CanvasDesigns_UserID ON CanvasDesigns(UserID);

-- Index for searching by name
CREATE INDEX IX_CanvasDesigns_DesignName ON CanvasDesigns(DesignName);
```

## 3. Querying JSON Data in MSSQL

Using `JSON_VALUE` and `JSON_QUERY`, you can extract specific data from the `ContentJSON` column without loading the whole string.

### Example: Find designs containing a specific layer name
```sql
SELECT DesignName 
FROM CanvasDesigns
WHERE EXISTS (
    SELECT 1 
    FROM OPENJSON(ContentJSON, '$.elements') 
    WITH (layerName NVARCHAR(100) '$.layerName')
    WHERE layerName = 'Logo'
);
```

## 4. Best Practices
1. **JSON Compression**: Since `NVARCHAR(MAX)` can store up to 2GB, large designs are fine. However, if space is a concern, consider Brotli/Gzip compression at the application layer before storage.
2. **CDN for Thumbnails**: Store the large Base64 strings (images) in a Cloud Storage (like Azure Blob Storage) and store only the URL in `ThumbnailUrl`.
3. **Soft Deletes**: Consider adding a `IsDeleted` bit column to prevent accidental data loss.
